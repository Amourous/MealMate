# Project Glossary: MealMate

This glossary defines technical terms, acronyms, and project-specific nomenclature used throughout the MealMate documentation and source code.

## A
- **API (Application Programming Interface)**: A set of protocols and tools for building software and applications.
- **Authentication**: The process of verifying the identity of a user or system.

## B
- **Backend**: The server-side part of an application, responsible for data processing and storage.
- **Better-SQLite3**: A high-performance, synchronous SQLite library for Node.js.

## C
- **CRUD (Create, Read, Update, Delete)**: The four basic functions of persistent storage.

## D
- **Dietary Filter**: A system feature that allows users to narrow down recipe choices based on dietary needs (e.g., Vegan, Vegetarian).
- **Docker**: A platform for developing, shipping, and running applications in containers.

## F
- **Frontend**: The client-side part of an application that users interact with directly (built with React).
- **Functional Requirement (FR)**: A requirement that specifies a function that a system or system component must be able to perform.

## G
- **Gantt Chart**: A bar chart that illustrates a project schedule, showing tasks and milestones over time.
- **Grocery Aggregation**: The process of summing ingredient quantities from multiple planned recipes into a single shopping list.

## J
- **JWT (JSON Web Token)**: A compact, URL-safe means of representing claims to be transferred between two parties, used for secure session authentication.

## L
- **LocalStorage**: A web storage object that allows data to be stored persistently in the user's browser for offline use.

## M
- **Milestone**: A significant event or stage in a project's timeline (e.g., "QA Complete").
- **MoSCoW (Must, Should, Could, Won't)**: A prioritization technique used in management and business analysis to reach a common understanding with stakeholders on the importance of each requirement.
- **MVP (Minimum Viable Product)**: A version of a product with just enough features to be usable by early customers who can then provide feedback for future development.

## N
- **Non-Functional Requirement (NFR)**: A requirement that specifies criteria that can be used to judge the operation of a system, rather than specific behaviors (e.g., Performance, Security).
- **Normalization (Ingredient Matching)**: The process of cleaning and standardizing ingredient names (trimming, case-insensitivity) to ensure accurate grocery list aggregation.

## P
- **Pantry Manager**: A feature that tracks a user's current kitchen inventory and integrates with the grocery list to avoid redundant shopping.
- **Postman**: An API platform for developers to design, build, test, and iterate their APIs.

## R
- **React**: A popular JavaScript library for building user interfaces.
- **RTF (Rich Text Format)**: A common file format for cross-platform document exchange (used for the project Abstract).

## S
- **SQLite**: A lightweight, disk-based database that doesn't require a separate server process.
- **Supertest**: A library for testing Node.js HTTP servers.

## T
- **Traceability Matrix**: A document that co-relates any two baseline documents that require a many-to-many relationship to determine the completeness of the relationship (e.g., Requirements to Tests).

## U
- **UML (Unified Modeling Language)**: A standardized modeling language used to visualize the design of a system.
- **Unit Testing**: A software testing method by which individual units of source code are tested to determine whether they are fit for use.

## V
- **Vite**: A fast build tool and development server for modern web projects.
- **Vitest**: A Vite-native unit testing framework.
